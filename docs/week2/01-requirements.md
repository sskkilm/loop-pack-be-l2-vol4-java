```
[유저 스토리]

- 특정 브랜드의 상품만 골라서 볼 수 있다.
- 최신 등록 순, 가격 낮은 순, 좋아요 많은 순으로 정렬해서 볼 수 있다.
- 페이지 단위로 볼 수 있다.
- 상품 이름, 설명, 가격, 브랜드 이름, 품절 여부, 좋아요 수 목록을 확인할 수 있다.

2. 브랜드 미지정 시 전체 브랜드 대상 조회
3. 정렬 기준 미지정 시 최신 등록 순 적용
4. 페이지 번호 미지정 시 0
5. 페이지당 개수 미지정 시 20
6. 상품 목록 반환
```

## 4. 주문 (Order)

### 🧑‍💻 대고객

#### O-01. 주문 요청

```
[유저 스토리]
- 사용자는 원하는 상품을 주문할 수 있다.
- 한 상품을 여러개 주문할 수 있다.
- 여러 상품을 한 번에 주문할 수 있다. 

[기능 흐름]
1. 로그인 사용자만 가능
2. 주문 항목 별 상품과 수량 입력 여부 확인
3. 미입력 시 필수 항목 누락 안내
4. 모든 상품 존재 여부 판단
5. 없는 상품이 포함된 경우 상품을 찾을 수 없어 주문이 실패함을 안내
6. 모든 상품의 재고가 주문 수량 이상인지 확인
7. 재고가 부족한 상품이 있으면 특정 상품의 재고가 부족해 주문이 실패함을 안내
8. 각 상품 재고 차감
9. 주문 저장
```

#### O-02. 유저의 주문 목록 조회

```
[유저 스토리]
- 사용자는 본인의 주문 목록을 조회할 수 있다.
- 기간을 지정하여 해당 기간의 주문만 조회할 수 있다.
- 주문 항목 별 상품 이름 · 가격 · 수량, 총 주문 금액, 주문 일시 목록을 확인할 수 있다.

[기능 흐름]
1. 로그인 사용자만 가능
2. 기간을 지정해 주문 목록 조회
3. 기간 미지정 시 전체 주문 목록 조회
4. 주문 목록 반환
```

> 비기능적 요구사항

1. 상품 좋아요 등록, 취소 기능은 같은 요청을 여러 번 해도 결과가 달라지지 않아야 한다.
2. 상품 좋아요 수는 등록, 취소가 동시에 실행되어도 정합성이 보장되어야 하며, 좋아요 수가 0 미만이 되어서는 안 된다.
3. 상품 좋아요 등록과 좋아요 수 증가, 그리고 좋아요 취소와 좋아요 수 감소는 각각 전부 실행되거나 전부 실행되지 않아야 한다.
4. 주문 시 상품 재고 차감과 주문 저장은 전부 실행되거나 전부 실행되지 않아야 한다.
5. 주문 시 차감되는 상품의 재고는 동시에 실행되어도 정합성이 보장되어야 하며, 재고가 0 미만이 되어서는 안 된다.
