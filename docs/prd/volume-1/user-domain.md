# 사용자(User) 도메인 PRD

> 1주차 학습 퀘스트 — 회원가입 / 내 정보 조회 / 비밀번호 수정 tdd로 구현하기
>
> 발제 요구사항 + `loopers-1-week.md` 를 취합한 내용이다.

---

## 0. 요구사항

### 회원가입
- 필요 정보: { 로그인 ID, 비밀번호, 이름, 생년월일, 이메일, 성별 }
- 이미 가입된 로그인 ID 로는 가입이 불가능함
- 각 정보는 포맷에 맞는 검증을 거친다
  - 로그인 ID: 영문/숫자로 구성된 1~10자
  - 이메일: `xx@yy.zz` 패턴
  - 생년월일: `yyyy-MM-dd` 포맷
  - 성별: `MALE` / `FEMALE` 두 값 중 하나로 필수 입력. 누락 또는 허용 값 외의 값이면 `400 Bad Request`
- 비밀번호는 암호화해 저장하며, 아래 규칙을 따름
  1. 8~16자의 영문 대소문자, 숫자, 특수문자만 가능
  2. 생년월일은 비밀번호 내에 포함될 수 없음

이후, 유저 정보가 필요한 모든 요청은 아래 헤더를 통해 요청한다.
- `X-Loopers-LoginId` : 로그인 ID
- `X-Loopers-LoginPw` : 비밀번호

### 내 정보 조회
- 반환 정보: { 로그인 ID, 이름, 생년월일, 이메일 }
- 로그인 ID 는 영문과 숫자만 허용
- 이름은 마지막 글자를 마스킹해 반환
  - 마스킹 문자는 `*` 로 통일
- 존재하지 않는 회원을 조회하면 `404 Not Found`
- Repository 조회의 미존재 케이스는 `Optional` 반환 컨벤션을 따른다

### 비밀번호 수정
- 필요 정보: { 기존 비밀번호, 새 비밀번호 }
- 비밀번호 RULE 을 따르되, 현재 비밀번호는 사용할 수 없다.

비밀번호 RULE
- 영문 대/소문자, 숫자, 특수문자 사용 가능
- 생년월일 사용 불가
